# CSS 动画原理：从 transition 到 @keyframes

作为前端学习的一部分，深入了解了 CSS 动画的工作原理。动画不只是让页面"动起来"，更是提升用户体验的重要工具。

## Transition 过渡

`transition` 是最简单的动画方式，它描述属性从一个状态到另一个状态的变化过程。

```css
.button {
  background: #2A7F6F;
  transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);
}

.button:hover {
  background: #35998A;
  transform: translateY(-2px);
  box-shadow: 0 4px 12px rgba(42, 127, 111, 0.3);
}
```

### 贝塞尔曲线

贝塞尔曲线决定了动画的"感觉"：

- `ease` — 默认，先快后慢
- `ease-in-out` — 缓入缓出
- `cubic-bezier(0.175, 0.885, 0.32, 1.275)` — 弹性效果

## @keyframes 关键帧动画

比 transition 更强大，可以定义多个中间状态。

```css
@keyframes fadeInUp {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.card {
  animation: fadeInUp 0.6s ease-out both;
}
```

## 性能优化

> 只对 `transform` 和 `opacity` 做动画，它们由 GPU 加速，不会触发重排。

### 应该避免的属性

- `width`, `height` — 触发重排
- `margin`, `padding` — 触发重排
- `top`, `left` — 触发重排

### 推荐使用的属性

- `transform: translate()` — GPU 加速
- `opacity` — GPU 加速
- `filter` — GPU 加速
